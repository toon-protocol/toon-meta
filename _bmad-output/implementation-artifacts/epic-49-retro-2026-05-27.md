## What Went Well

### 1. Revenue Loop Proven End-to-End

`v0.1-pilot-readiness.md` says GO. The full loop closes: Foreign ToonClient → SOCKS5 → ATOR → `.anyone` HS → apex connector → town relay → DVM → Arweave txid in ILP FULFILL → earnings credited. EVM settlement on Akash-hosted Anvil. This is the product working as designed on real infrastructure.

### 2. The 49.5 Architecture Pivot

Moving from Akash-hosted foreign pod (49.4's binding constraint) to local Docker on an isolated `e2e-client-net` network was the right call. Akash provider quality (4 consecutive failures in 49.4) was the constraint, not the story logic. The pivot preserved NFR5 (`.anyone` transport invariant — client and apex can only reach each other via ATOR) while eliminating provider variability. The pivot felt right because it was right.

### 3. A13' (Two-Pass Review) Applied and Proved Its Value

49.1 received two review passes (86 total patches). Pass 2 caught real defects: `probePortFree` polarity inverted (would pass when port was occupied), dead `routeOverride` fallback that silently guaranteed `PEER_NOT_NEGOTIATED`, Docker bridge IPAM multi-gateway concatenation on dual-stack hosts. These are not style findings. The two-pass commitment for surgery-class stories is validated.

### 4. A14' (Re-scope) Applied: Minimal Viable Gate Delivered

The whole epic re-scope was A14' at scale. Instead of Cloudflare + Grafana + custom domain telemetry infrastructure, we delivered a focused E2E gate proving the actual revenue loop. Right-sized for pilot.

### 5. Key Technical Discoveries That Save Future Pain

- **`TurboFactory.unauthenticated()` is read-only** — `uploadFile` undefined at runtime. Use `TurboFactory.authenticated({ privateKey: ephemeralJwk })` for free-tier uploads (≤100KB). Now documented.
- **Connector `routes: []` causes F02 before `localDelivery`** — explicit self-route entry required in connector.yaml.
- **Sub-path C (public ATOR proxy, no in-pod connector) IS viable** for foreign pods that only need to dial `.anon` addresses, not host them.
- **`getPeers()` ≠ channel registry for foreign BTP clients** — use `getChannels()`. Fundamental connector architecture; now documented.

## Challenges

### 1. Every Story Was a First-Contact Story — Firefighting Was Structural

The epic was mostly firefighting. Every story hit a blocker not visible from the spec:
- 49.1: `getPeers()` semantics, dual-stack Docker bridge IPAM, multi-line JSON parsing
- 49.3: `ator-onion` mode was the wrong architecture (caught during deploy, not at spec time)
- 49.4: `assetCode` is a token contract address not `'USD'`; `recentClaims` not filtered by `testStart`
- 49.5: `TurboFactory.unauthenticated()` has no `uploadFile`; `routes: []` causes F02

Unlike Epic 48 where a scaffold story amortized complexity across downstream stories, Epic 49 had no downstream — every story was frontier. The patch volume (avg 36/story vs. 10.4 in Epic 48) reflects real defects caught by review, not process bloat.

**Systemic observation:** The spec is written against what we *think* the system does, not what we've *verified* it does. For stories touching new infrastructure, a lightweight system-probe step before spec-writing would surface some of these surprises earlier and cheaper.

### 2. Akash Provider Quality as Binding Constraint

49.4's 4-consecutive-provider failure cascade (Solana seccomp `io_uring_setup` + cross-provider HTTPS instability) blocked a live GREEN gate capture. The green gate was inherited by 49.5. For any story that requires live Akash infrastructure for a gate run, provider quality is an uncontrolled variable — plan for re-runs and pre-vet providers.

## Key Insights

1. **Firefighting is unavoidable on frontier stories — the mitigation is architecture, not process.** When every story is first-contact, the pivot instinct (49.5) matters more than the spec quality.

2. **Carry-forwards rot unless they become story ACs.** A10'/A11'/A12' have proven this across three epics. Story 50.0 closes this failure mode structurally.

3. **`deferred-work.md` needs a triage cadence.** It's growing faster than it's shrinking. Before each epic, the items that affect real operators should be separated from the nice-to-have engineering cleanup.

4. **The local-Docker isolation pattern is the right E2E architecture.** Separate networks, ATOR as the only bridge between client and apex, Docker for container lifecycle. Eliminates Akash variability while preserving transport invariants. This is the template for Epic 50's gate.

5. **Two-pass review for surgery-class stories: keep the commitment.** Two data points (48.4, 48.6) from Epic 48, confirmed again at 49.1. Pass 2 catches real defects that pass 1 misses under time pressure.

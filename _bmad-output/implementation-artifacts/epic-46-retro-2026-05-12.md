### The Yaml-First Ordering Invariant Held Under Pressure

The architectural rule from Epic 46 planning — `nodes.yaml` write happens BEFORE connector registration (step 3 < step 6) — held across three stories and a multi-round review. Story 46.2's code review explicitly considered flipping the rollback order (Decision D2) and rejected it. The rule landed in code comments at the wiring point (`packages/townhouse/src/reconciler.ts` and `packages/townhouse/src/api/routes/nodes-lifecycle.ts`) where future maintainers will see it before they reach the spec.

### State-Machine Rollback Tests Caught What Mocks Cannot

Story 46.2 AC#3 — *"rollback completes byte-identical to pre-add state"* — landed as a table-driven test injecting failures at each of 6 pipeline steps. 27 tests in `nodes-lifecycle.test.ts`, all green, zero regressions. This is the single most important test artifact in the epic; it ensures that any future implementer of the pipeline cannot regress the rollback contract without the suite catching it.

## Challenges

### 1. Release Pipeline ↔ Epic Merge Ordering (Finding C — Structural)

The rc5 tarball shipped *before* Epic 46 merged. The gate hit `POST /api/nodes` → 404 because the API didn't exist in the published image. Required cutting rc6 to validate. The coupling between release cadence and epic close-out wasn't structural — it relied on manual coordination.

**Why it matters:** A pilot operator running `npm install -g @toon-protocol/townhouse@0.1.0-rc5 && townhouse hs up && townhouse node add town` would have hit a 404 with no recovery path. The gate caught it before any operator did, but only because the gate happened to run against the published artifact rather than local dev.

**Root cause:** The publish workflow can be triggered against any commit; there is no structural assertion that the tarball under test contains the epic's merge SHA before the epic is marked done.

### 2. Compose-Template ↔ CLI Contract (Findings B, B.1, D–O)

Ten of the 14 findings sit at the seam between the HS compose template (`packages/townhouse/compose/townhouse-hs.yml`), the CLI's env-export logic (`handleHsUp` / `handleHsDown`), and the orchestrator's profile-aware container creation. Finding B.1 was textbook: PR #50 added `${TOWNHOUSE_HOME}` interpolation and exported it in `handleHsUp` but forgot to export it in `handleHsDown` — the fix introduced a teardown regression.

**Root cause:** The compose template is a text file the CLI substitutes into; the contract between "vars compose references" and "vars CLI exports" is implicit, not enforced.

### 3. Cross-Repo Dependency Drift (Finding A)

The rc5 manifest pinned connector v3.4.1, which predates the `/admin/hs-hostname` endpoint shipped in connector v3.5.0 (toon-protocol/connector#59, Story 44.1). The publish workflow's manifest generation doesn't track the connector pin actively — the version was set when rc5 was cut and stayed there even as Town's source code began requiring v3.5.0+ endpoints.

### 4. Gate Iteration Cost (3 Runs to Green)

Each gate run took 8–12 minutes wall-clock. Three runs were needed:
1. First run — uncovered Findings A + B (apex didn't boot)
2. Second run (post-PR #50 merge) — uncovered Findings B.1 + C (apex boots, but `/api/nodes` 404)
3. Third run (against rc6) — 5/5 PASS

The iteration loop was slower than ideal because findings were release-cycle-shaped. A faster inner-loop check (sub-minute compose-config-parse, local-image dry-run) would have caught Findings B/B.1/D–I/L/M before the full gate ran.

## Key Insights

1. **The live gate is the load-bearing safety net.** Code review caught logic errors; the gate caught integration gaps. Both are required, but only the gate sees what the operator sees. The Epic 45 retro's A4 bet was correct; Epic 46 confirms it.

2. **Cross-repo coupling has its own failure modes.** Findings A (connector pin), C (release ordering), and Q (upstream defect) are all release-coordination class — invisible to unit suites by construction. Multi-repo systems need multi-repo discipline.

3. **The compose-template seam is structurally under-protected.** 10 of 14 findings were compose ↔ CLI ↔ orchestrator. A static contract test would have caught most of them in sub-minute feedback instead of in the 8–12 min gate.

4. **Process commitments compound.** Five Epic-45-retro action items all landed in Epic 46. The compounding effect is real: dated review findings (A1) made the gate (A4) evidence-bearing, which made cross-repo findings (Finding C) trackable, which justified the gate-sharpening commitments below.

## Action Items

### A1' — Tarball-SHA Gate Guard (counters Finding C)

**What:** Add to every `*-live-e2e-gate-*` story's Close-Out Checklist: *"Verify the tarball under test contains this epic's merge SHA before declaring PASS."* Mechanically: parse the tarball's `package.json` build metadata (or the publish workflow run's input SHA) and assert it matches `git rev-parse <epic-merge-branch>`.
**Owner:** Jonathan (process); Amelia (story template change)
**Deadline:** before Story 47.5 (Epic 47's gate) is created
**Success criteria:** No epic's gate story flips to `done` against a tarball that predates the epic's merge.

### A2' — Cross-Repo Connector Pin Source-of-Truth (counters Finding A)

**What:** Each epic doc with connector contract dependencies declares the minimum connector version in the epic plan; the publish workflow's manifest generation reads from a single source. Today the connector pin lives only in the publish workflow's default input — there is no automated assertion that source code's `getEarnings()` / `getHsHostname()` / `removePeer()` calls match the pinned version.
**Owner:** Winston (architect — picks the SoT location); Jonathan (publish workflow integration)
**Deadline:** before Epic 47 starts (Story 47.1 depends on connector v3.3.3+; we just shipped v3.6.2 so this is already satisfied for 47, but the structural fix prevents future Finding-A regressions)
**Success criteria:** A grep for "connector v3" in the codebase returns exactly one location of truth, referenced by publish workflow input default and Epic 47's plan.

### A3' — Compose-Template ↔ CLI Contract Test (counters Findings B, B.1, D–O)

**What:** Structural unit test that every `${VAR}` reference in `packages/townhouse/compose/townhouse-hs.yml` (and any sibling compose files) is exported in BOTH `handleHsUp` AND `handleHsDown` (and any future analogue). The test parses the compose YAML, extracts variable references, and asserts each appears in the CLI's env-export list.
**Owner:** Amelia (write the test)
**Deadline:** before Epic 47 starts — this is sub-minute and unblocks the dry-run pattern
**Success criteria:** `pnpm --filter @toon-protocol/townhouse test compose-env-contract` passes against current main; running the test against the B.1 regression (TOWNHOUSE_HOME exported in handleHsUp but not handleHsDown) **fails**.

### A4' — Pre-Merge Gate Dry-Run Pattern (counters gate iteration cost)

**What:** Document the dry-run path in the gate-story template: how to build a local image, materialize a local manifest, and run the gate against unpublished code with a `NOT-RELEASE-VALIDATED` banner. The dry-run is for inner-loop development (catch B.1-class regressions in story dev, not at release). The full gate against the cut tarball remains the gate of record for `done`.
**Owner:** Amelia (write the dry-run doc as a section in the gate-runbook, or a new `gate-runbook.md`)
**Deadline:** before Story 47.5 is implemented
**Success criteria:** A new contributor can run the gate dry-run against unpublished code in <5 minutes from the runbook alone.

### Deferred / Carry-Forward (not Epic 47-blocking)

5. **Finding K (dormant) — ATOR transport mode loopback fix.** Not an Epic 47 blocker. Tag for Epic 49 (telemetry/transport) or whoever lands ATOR HS configuration first. Same fix shape as Finding G.
6. **46.3 deferred items (~10) — sweep PR opportunity.** Low-priority polish: `getPeers()` error classification, `AbortController` not torn down on success, no SIGINT cleanup during `confirmInteractive`, unbounded id length, ANSI passthrough in stderr, yaml-read 500 leaks filesystem path, `'disconnected'` collapses missing-peer states, `parseArgs({strict: false})` typos, DELETE-id regex duplicated, STAGE_LABELS visual-order mismatch.

## Challenges

### 1. Review Step Skipped on Two Stories

Stories 45.1 and 45.4 went from `ready-for-dev` to `done` without a logged code review. Both required significant post-merge patching (4 fix PRs for 45.1, 10 bugs patched post-merge for 45.4). Story 45.4's `### Review Findings` was still "Pending review" at ship time.

**Root cause:** The review step requires a deliberate context switch that's easy to skip when the implementation feels done, especially after the intensive 3-round review of 45.3.

**Pattern:** Stories that skipped review needed post-merge patches. Stories that went through full review shipped with blockers caught pre-merge.

### 2. Regex/Template Logic Without Golden-Input Tests

Two separate regex bugs across the epic:
- 45.2: Template substitution regex introduced during round-1 patching failed 100% of the time — caught in round 2.
- 45.3: `surfaceComposeFailure` broke after the first match — caught in review.

Both had the same gap: no unit tests using realistic input strings. Review compensated, but at the cost of extra review rounds.

### 3. Skip-Gated Integration Tests Hiding Real Crashes

45.3's integration test for `waitForHsHostname` was skip-gated (`RUN_DOCKER_INTEGRATION`). It passed `undefined as never` for config — a `TypeError` on first real run. Caught in round-3 review, not by the test suite.

### 4. Live E2E Testing Was Ad-Hoc, Not Structural

The 10-bug catch in `572544f` happened because live testing was run after merge. If it had been a mandatory story step, those bugs would have been patched before the epic closed. The activity was valuable; its placement was not.

### 5. ~22 Deferred Items Carrying into Epic 46

Several deferred items from 45.3 are load-bearing for Epic 46's `node add` pipeline:
- `activeNodes` stale state on error
- No rollback when `waitForHsHostname` times out
- `downHs` not idempotent on "nothing running"
- `surfaceComposeFailure` hardcoded `townhouse-hs-` prefix (silent for peer node failures)
- Non-503/non-200 statuses retried silently for 120s

---

## Key Insights

1. **The review process works — when you use it.** The failures in this epic were proportional to where review was skipped. This is the most load-bearing insight of the epic.

2. **Live e2e testing catches a different class of bug than code review.** Code review finds: logic errors, trust-boundary gaps, API misuse. Live testing finds: fixture mismatches, behavioral gaps in real infrastructure, UX issues. Both are required.

3. **Patch-introduced regressions need the same scrutiny as original code.** The round-1 regex patch in 45.2 that failed 100% of the time was introduced *during* the review process. Review passes need to cover patches, not just the original diff.

4. **Regex and template logic without realistic-input tests is a review tax.** Every such gap defers bug discovery to the code reviewer. Cheap to fix upfront; expensive to find in round 2.

## Action Items (Process — applies to all future epics)

### A1 — Review Gate
**What:** Update the `bmad-create-story` story template so `### Review Findings` has a dated placeholder that must be replaced (not blank). Add one line to the close-out checklist: "Verify `### Review Findings` contains a dated entry before flipping sprint-status to `done`."
**Owner:** Jonathan
**Success criteria:** No story flips to `done` with a blank or "Pending review" `### Review Findings` section.

### A2 — Regex/Template Golden-Input Tests
**What:** For any new regex or template substitution logic, the story AC must include at least one unit test using a real-world input string. Story template checklist gains: "Does this story contain regex or template logic? If yes, golden-input test required."
**Owner:** Jonathan
**Success criteria:** Epic 46 stories with pattern matching have at least one test using a realistic input.

### A3 — Skip-Gated Test Convention
**What:** Any `skipIf` / `describe.skip` / `RUN_*`-gated test must have a comment: `// Gate: <condition>. Run this before marking story done.` Story close-out checklist gains: "Were any skip-gated tests on the critical path un-gated and run?"
**Owner:** Jonathan
**Success criteria:** No story flips to `done` with an un-exercised skip-gated test on the critical path.

### A4 — Live E2E Gate Story (Every Epic)
**What:** Every epic gets a final story: `X.N — Live E2E Gate`. AC: run the epic's primary user journey end-to-end against real infrastructure, document findings in the story file, patch any bugs before flipping epic to `done`. Precedent: Story 8.6.
**Owner:** Jonathan (add to epic planning convention; Story 46 plan should include this before dev starts)
**Success criteria:** Epic 46 has a `46.N — Live E2E Gate` story in the sprint plan.

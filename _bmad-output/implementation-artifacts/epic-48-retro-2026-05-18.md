## What Went Well

### The Slot-Mount Scaffold Pattern from 48.1 Paid Dividends Across Six Downstream Stories

Story 48.1 established `<App />` with `HeaderSlot` / `MainSlot` / `FooterSlot` and exposed a single 2-second refresh tick. Every downstream story (48.2 ApexStrip/PeerTable, 48.3 Badge, 48.4 ActivityTicker + ActivityOverlay) mounted via those slots with **zero surgery to 48.1's scaffold**. No story required re-opening the wire shape, the refresh tick, or the TTY-detection ladder.

The pattern's real win was in Story 48.3 — 1 patch / 1 deferred / 11 dismissed across 13 raw findings — the cleanest story of the epic. The badge slotted into the existing hero region, consumed the existing wire, and added no new copy tokens. That's what a load-bearing scaffold buys downstream.

### Empty-State Copy Library as Merge Gate (NFR19 Held)

UX-DR2 was enforced as an AC on Story 48.1: every zero, wait, and loading state has explicit copy (no `if (n === 0) return ''`). Sally's verbatim audit on Story 48.7's gate run validated every rendered string in Tests 1, 2, and 3-A against the locked library — **all matches exact**. The bonus discovery (`Last refresh failed — retrying.` rendered verbatim against the E2 dev-image's missing earnings route) means the failure-state copy contract was also exercised under real conditions, not just unit tests.

Sally signed both UX-DR2 (2026-05-18) and UX-DR3 (2026-05-14, pre-confirmed 2026-05-18) verbatim. NFR19's merge-gate teeth held.

### Two-Pass Code Review Emerged as a Recognized Pattern

Two stories — 48.4 and 48.6 — went through a second review pass and both passes found real defects:

- **48.4 pass 2:** Q1 (loading-phase keypress guard — pressing `[a]` during `phase: 'loading'` opened an empty overlay) + Q2 (resize scroll-reconcile useEffect — terminal-resize during overlay-open left scroll position above `maxScroll`).
- **48.6 pass 2:** P4 (empty-flag env fallback — `TOWNHOUSE_SATS_PER_USDC=''` fell through to default rate silently) + P5 (ZodError one-liner — multi-line `ZodError.message` corrupted CLI error output).

Neither defect would have shipped if we'd stopped at pass 1. The pattern now has two data points — codified in A13' below.

### The Live E2E Gate Caught Three Classes of Environmental Drift That Snapshots Could Not

48.7 took five gate runs to clear. Runs 1–4 hit:
- **E1:** dist bundle bug — duplicate `createRequire` import + wrong package.json path in published artifact (the runbook's `/tmp/runbook-48-7/mount-tui.mjs` workaround patched it locally)

## Challenges

### 1. Five Gate Runs Required for 48.7 (vs. 3 for Epic 46/47)

Run 1 hit E1 (dist bundle bug). Run 2 hit E2 (stale image-manifest in CI). Run 3 hit E3 (wrong connector version pinned). Runs 4 and 5 cleared.

None of these were product defects. All three were drift between *what we built locally* and *what got tested.* Two of the three would have been caught by A6' (connector publish smoke) if it had landed. The third (E1, dist bundle) is a new shape — a npm-publish-pipeline issue, not a connector-coupling issue.

**Iteration cost:** Each run was ~60–90 seconds; the cost was the *investigation* time between runs (figuring out which version mismatch caused which symptom). Total elapsed: roughly 4–6 hours, vs. Epic 47's 3 runs which cleared same-day.

### 2. Dev-Image Drift Is the Connective Tissue

Jonathan's insight during the retro: **we were testing the 48.7 manual runbook against a fiction.** The `epic-47-local` image is a locally-built tag missing Epic 47's earnings route, so the TUI's `useEarnings` hook fell through to `fetch_failed` and rendered the "Last refresh failed — retrying." stale-data hint. We *verified* the copy library because of this — but only by accident.

The same drift is what made E2 and E3 possible: the CI manifest and the published images don't share a single source of truth. There's no mechanical check that says *the image you're testing against has the digest the manifest pins.*

This is connective tissue between three separate Epic 47 commitments:
- **A1'** (rc7 publish + production gate) — still open. Epic 47 gates also ran against `epic-47-local`.
- **A6'** (publish smoke) — addresses the connector side of the drift.
- **A11'** (compose contract — formerly A5') — addresses the config-emit side of the drift.

A9' below makes the drift mechanical for *any* image, not just the connector.

### 3. Story 48.5 Drift-Class Bug Found in Pass 1, Could Have Been Earlier

48.5's BLOCKER finding — `townhouse health` always returns `connector: unreachable` against a real apex because `probeConnector` called `adminClient.getHealth()` (rich shape) against the slim `/health` endpoint at `:9401` — was a *real* defect caught by Edge Case Hunter on first review pass. Mocks in `drill-commands.test.ts` happened to match the rich shape so unit tests stayed green.

This is the *fourth* defect class in 4 epics where unit-test mocks accidentally validated themselves rather than the real wire shape. (Epic 46's BUG-1 chainProviders, Epic 47's BUG-2 ClaimReceiver, Epic 47's wire-shape `status` field, now Epic 48's slim-vs-rich `/health` shape.) Worth naming: **mock-shape drift is now its own category.**

### 4. Story 49.5 Over-Engineered for Pilot Scale (Caught at Retro)

Jonathan questioned why 49.5's receiver server reaches for Cloudflare + custom domain + Grafana for a 30-day pilot with 5 operators (~20 total weekly records). The honest answer: Cloudflare buys NFR5 (IP-strip-at-edge) as a structural guarantee rather than a "we promise not to log" pinky-swear, and Grafana buys NFR6 (server-outage detection before dataset corruption).

For pilot scale, NFR5 can be satisfied by nginx + `proxy_set_header` removal (5 lines), and NFR6 by a manual `tail -f` on `pings.jsonl` (1 line). The Cloudflare/Grafana stack is correctly sized for v1.0 public launch, not v0.1 pilot. A14' captures the re-scope.

## Key Insights

1. **The Live E2E gate is the project's identity now.** Three consecutive epics where it found things unit tests couldn't. Jonathan picked "Live E2E gate as second-stage filter" as the win to repeat verbatim in Epic 49. It is no longer up for debate — every epic ships a `*-live-e2e-gate-*` story.

2. **Dev-image drift is the meta-defect.** It's the connective tissue under three Epic 47 commitments (A1', A5', A6'). Mechanizing it (A9' below) closes a class of defect that has bitten 48.7 (5 gate runs), 47.5 (3 gate runs), 46.4 (multiple findings) — and will bite 49.7 if not fixed.

3. **The slot-mount scaffold is a reusable architectural pattern.** Epic 49 telemetry has parallels — `townhouse telemetry on|off|status` (49.4) is structurally like a drill subcommand (48.5). The slot pattern should be the default mental model for "add a new CLI verb without surgery." Worth naming explicitly in 49.4's Dev Notes.

4. **Two-pass review for surgery-class stories should be pre-committed, not emergent.** 48.4 and 48.6 both yielded real defects on pass 2. Codifying this in the story template (A13') prevents the "shipped after pass 1" failure mode.

5. **Action-item follow-through is structurally broken.** A5' + A6' + A7' are all overdue. The common factor: none of them touched Epic 48's surface area, so they fell off. Solution: either deadline them to a specific story start (e.g., A11' "before Story 49.5 starts") or de-scope explicitly. Vague carry-forwards rot.

6. **Mock-shape drift is its own defect class.** Four epics in a row, a unit-test mock has validated itself rather than the real wire shape (chainProviders, ClaimReceiver, status field, slim-vs-rich /health). A contract-canary-style test against the real downstream service's documented schema would catch this — but it's expensive to build for every endpoint. Worth thinking about during Epic 49 planning.

## Action Items

### A9' — Dev Image Manifest Pinning + Drift Detection (NEW)

**What:** Every story that runs against a local image declares the image-manifest digest it tested against in its Review Findings. A pre-flight check in the gate runbook asserts the digest matches `packages/townhouse/dist/image-manifest.json`. If not, the gate **refuses to run** with a clear error.

**Why:** Jonathan's insight at retro: 48.7 tested the TUI against `epic-47-local`, a locally-built tag missing Epic 47's earnings route. Failure-state copy was verified by accident. Epic 49's gate WILL reproduce this drift class if 49.7 is built against a similarly-stale image.

**How to apply:** Sub-15-line shell guard in `scripts/rerun-earnings-gate.sh` (and equivalent for telemetry gate). Checks `docker inspect <image> --format '{{.Id}}'` against the manifest's pinned digest. Fails fast: `Image <tag> has digest X but image-manifest pins Y — refusing to run gate against drift.`

**Owner:** Amelia (write the guard); Winston (define manifest contract)
**Deadline:** **before Story 49.7 starts** (or before 49.1 if convenient)
**Success criteria:** Replaying 48.7 run 1 against `epic-47-local` fails the pre-flight; running against published rc7 passes.

### A10' — Connector × Townhouse Publish Smoke (carry forward A6', now overdue 2 epics)

**Why:** Overdue two epics. Epic 49 telemetry posts depend on the connector being structurally trustworthy — if connector v3.6.4 silently breaks `/admin/earnings.json`, telemetry pings carry garbage for a full week before the pilot day-30 query catches it.

**Owner:** Winston (architect picks repo); Jonathan (workflow integration)

## Best-practice highlights

1. **Type-shape compile-time guard (T-8)** — the `satisfies IssueClaimParams` pattern turns an AC-10 type-boundary assertion into a compile-time guard. Any future change that removes `chainRecipient` or `senderPubkey` from the interface breaks `tsc`, not runtime. This is the best-in-class pattern for codifying type contracts in test files.

2. **Defect-reproduction in RED form** — T-10 reproduces the original Story 12.8 session-3 defect at the unit level: `expect(arg.recipient).not.toBe(SENDER_PUBKEY)`. The negative assertion is named with the exact defect semantics so a reader grasps what regression is being guarded without re-reading the story.

3. **Guardrail encoding as tests (T-13)** — the sender→channel sticky-binding guardrail (8.3) is captured as `reserveSpy.mock.calls[0][0].senderPubkey === SENDER_PUBKEY && !== FIXTURE_EVM_RECIPIENT`. Spec-as-test for a non-functional constraint.

4. **Tag-order independence (T-7-AC3)** — explicit customRumor with `chain-recipient` placed BEFORE `swap-from` / `swap-to` proves AC-3's "parse by name" requirement, pinning the implementation against a future ordering-dependent refactor.

5. **Contract duality (T-2d)** — the unknown-chain test exercises BOTH halves of the entry-guard / validator composition (`INVALID_STATE` for empty, pass-through for non-empty opaque), preventing a future collapse of the two guards from regressing the fall-through.

### Nice-to-have (deferred, not blocking)

1. **`FIXTURE_EVM_RECIPIENT` duplication across 4 files.** Each file declares its own local `FIXTURE_EVM_RECIPIENT = '0x' + '11'.repeat(20)`. Consolidating into a shared `packages/sdk/src/test-fixtures.ts` would cut duplication but the story's Testing Standards explicitly permit per-file definition ("or a small shared `test-fixtures.ts` if it already exists in the package — check before creating one"), and no such file exists. Leaving as-is respects guardrail 8.5 (no new shared helper files).

2. **T-2d comment density.** The test body has a long multi-paragraph comment block before the first assertion. Valuable context, but a reader navigating to the assertion has to scroll. Consider a companion `## Unknown chain contract` note in the test file header in a future edit.

3. **Flat describe for Story 12.9 blocks.** Each file has a single `describe('Story 12.9 — …', …)` with flat `it(…)` entries. Nested describes per AC group (e.g., `describe('AC-8: missing/malformed tag', …)`) would help pre-failure triage from CI output, but the AC-in-title convention already addresses this.

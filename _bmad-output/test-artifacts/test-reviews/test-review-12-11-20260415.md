### Key Weaknesses

This is tracked honestly in the story's Dev Notes + Change Log and the blocker predates this story, but three P0/P1 gates staying RED on a story in `review` status is a real risk that warrants an explicit follow-up story (already flagged in the story's Completion Notes).

The ATDD offers both options as acceptable; picking "inline" means the probes cannot be re-run by a script from a future debugger or by CI — only copy-pasted by hand. ## Recommendations (Should Fix)

### 1. (Fixed in-place) AC-4 EXPOSE regex was over-permissive

**Severity**: P1 (High)
**Location**: `_bmad-output/test-artifacts/atdd-checklist-12-11.md` AC-4 probe
**Criterion**: Explicit Assertions
**Knowledge Base**: [test-quality.md](../../../../_bmad/tea/testarch/knowledge/test-quality.md)

**Issue Description**:
The probe `grep -E 'EXPOSE .*3000.*3100.*7100' docker/Dockerfile.sdk-e2e` would match `EXPOSE 1300 3000 3100 7100` — the exact failure mode the story forbids (AC-4 explicitly requires "no 1300"). `.*` is permissive across tokens, so a Dockerfile with the forbidden port could still pass this probe if the ports appeared in order elsewhere.

**Fix Applied**:

```bash
# Before
grep -E 'EXPOSE .*3000.*3100.*7100' docker/Dockerfile.sdk-e2e  # ❌ loose

# After
grep -E '^EXPOSE +3000 +3100 +7100 *$' docker/Dockerfile.sdk-e2e  # ✅ anchored
grep -c '1300' docker/Dockerfile.sdk-e2e  # ✅ must be 0
```

**Why This Matters**: AC-4 is P0 — any slack here defeats the story's core hygiene goal.

### 2. (Fixed in-place) AC-7 USER-before-CMD was asserted informally

**Severity**: P2 (Medium)
**Location**: `_bmad-output/test-artifacts/atdd-checklist-12-11.md` AC-7 probe
**Criterion**: Explicit Assertions

**Issue Description**:
The probe claimed "USER toon appears before the final CMD line" but only grepped for existence. Dockerfile semantics are order-dependent: a Dockerfile with `CMD [...]` then `USER toon` (wrong order, CMD runs as root) would pass the original probe.

**Fix Applied**:

```bash
awk '/^USER toon/{u=NR} /^CMD /{c=NR} END{exit !(u && c && u < c)}' docker/Dockerfile.sdk-e2e
```

This exits 0 only when both directives exist and USER precedes CMD.

---

### 3. (Fixed in-place) Missing runtime-artifact probe

**Severity**: P2 (Medium)
**Location**: `_bmad-output/test-artifacts/atdd-checklist-12-11.md` (new probe block added between AC-9 and AC-11)
**Criterion**: Runtime-Artifact Validation

**Issue Description**:
AC-3, AC-4, AC-5, AC-7 all assert Dockerfile **source text** via grep. A Dockerfile that passes all source probes could still ship an unwanted `attestation-server.js` via a transitive COPY (e.g., `COPY docker/dist/ /app/` would copy both entrypoint-sdk.js AND attestation-server.js in one line and pass every source probe). Source-only probes are necessary but not sufficient for behavior assertion on built images.

**Fix Applied**: added a new probe block using `docker run --rm --entrypoint sh toon:sdk-e2e -c ...` to assert:
- `/app/attestation-server.js` absent
- `/app/entrypoint-sdk.js` present
- `supervisord` not installed
- runtime uid/gid is `1001/1001` (USER toon is effective at runtime)

This closes the source→built-artifact gap and is the strongest behavioral assertion available without running the full SDK E2E suite.
